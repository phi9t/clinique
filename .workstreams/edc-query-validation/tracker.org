* Overview
:PROPERTIES:
:ID: edc-query-validation-overview
:STATUS: IN-PROGRESS
:OWNER: unassigned
:VALIDATION: RFC-0006 L0-L4
:ARTIFACTS: .workstreams/edc-query-validation/design.md
:END:

Build the first data-rich validation workstream for Clinique beyond the current
biostatistician suite: a draft-only EDC/data-management query agent. The workstream proves
workflow usefulness, not generic model accuracy, using historical EDC snapshots, query logs,
query resolution history, database-lock issue logs, edit-check rule history, and later silent
prospective evidence.

North-star question: does the agent improve data-cleaning outcomes without increasing
participant, data-integrity, blinding, privacy, or compliance risk?

** Global acceptance criteria
- [X] No write-back path to EDC or any system of record exists in this workstream for the local
      synthetic harness.
- [X] Every recommendation is traceable to source fields, rules, snapshot timestamps, tool/model
      versions, and human review status.
- [X] Retrospective replay uses only evidence available at the replay timestamp.
- [X] Offline and replay metrics include task labels and workflow/outcome labels.
- [ ] Silent-mode protocol is approved before any production-like run.
- [X] Ship gates explicitly include false-query burden, duplicate-query burden, privacy,
      blinding, and auditability.
